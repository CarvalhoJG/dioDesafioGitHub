SHA1
Secure Hash Algorithm/Algoritmo de Hash Seguro
Conjunto de funções hash criptograficas projetadas pela NSA(Agecia de Segurança Nascional dos EUA), que gera um conjunto de quarenta digitos chamados de characteres identificador. Cada vês que o arquivo é alterado um novo conjunto de characteres. 


echo 'conteudo' | git has -object --stdin
>fc31e91b26cf85a55e072476de7f263c89260eb1

Entendendo os comandos:

Imprimir o output e passar o "conteudo" para uma função do git

Calcula o valor do ID do objeto para um objeto com o tipo especificado com o conteúdo do arquivo nomeado

--stdin
Leia o objeto da entrada padrão em vez de um arquivo.

Blobs: 
É um objeto que armazena metadados sendo eles:
tipo do objeto, tamanho da string ou desse arquivo, contrabarra zero(\0) e o contudo de fato desse arquivo. Ela é responsável por motar a estrutura de onde esta localizado os arquivos.

echo -e 'blob 9\0conteudo' | openssl sha1
>fc31e91b26cf85a55e072476de7f263c89260eb1

Trees
As trees(arvores) armazenam os blobs, contem metadados e o contrabarra zero(\0) aponta para um blob que possiu um SHA1 e o nome do arquivo.

Os blobs tem o sha1 do arquivo, as tree apontam para os blobs que possuem um sha1 encriptado com os metadados das tree para caso ocorra alguma alteração no arquivo o qual a tree esta referindo(apontando) o sha1 desse metadados do blob sofrerá alteração no conjunto de caracteres e será possível ver a alteração do sha1 da tree na sequência
